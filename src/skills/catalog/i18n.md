# Internationalisation (i18n)

## Key principles

- Never hardcode user-facing strings — always use translation keys.
- Keep keys hierarchical: `auth.login.submit`, `errors.required`.
- Default locale: `en`. Fallback chain: locale → language → en.

## Date and number formatting

- Use `Intl.DateTimeFormat` and `Intl.NumberFormat` — never format manually.
- Always pass the locale explicitly; never rely on `process.env.LANG`.

## Pluralisation

- Use `Intl.PluralRules` or a library (e.g. `i18next`) — never `count === 1 ? 'item' : 'items'`.

## File layout

```
locales/
  en/common.json
  fr/common.json
```

Each JSON object: `{ "key": "translated string" }`.

## RTL support

- Set `dir="rtl"` on the root element when locale is Arabic, Hebrew, etc.
- Use logical CSS properties (`margin-inline-start`) not physical (`margin-left`).
